Dynamic-LOD

The amount of datasets in the world of Linked Data is growing each day. Yet, counting links and visualizing the relations between these datasets is one of the challenges which the community still struggles to solve. The creation of diagrams which visualize the Data Web with a rich level of meta-data is a hard task and a crucial challenge of any analysis is to provide accurate methods to analyze links between data. A second challenge is the freshness of such an analysis. Especially in the case of large amount of big datasets, which are hosted in a decentralized way, detecting and counting links requires a centralized index and methods that are scalable enough to keep up-to-date.

Dynamic-LOD uses Bloom filters to compare and count links between datasets. Using Dynamic-LOD you can add your dataset in our cloud, and after we analyze it, you can retrieve metadata using void:Linkset format.

How can I add dataset to Dynamic-LOD?

It's simple. You can add a dataset using our API, and you need to specify what's the file format. For instance, to add a file "http://myDomain.org/void.ttl" in turtle format, you must access:
http://vmdbpedia.informatik.uni-leipzig.de:9999/api?addDataset=http://myDomain.org/void.ttl&rdfFormat=ttl
The addDataset parameter accepts VoID, DCAT and DataID descriptions. Dynamic-LOD request the header "Accept: application/rdf+xml" in order to retrive data in RDF format. The rdfFormat parameter accept "nt","ttl", or "xmlrdf" and is the format of the addDataset parameter.

How to get Dynamic-LOD?

Dynamic-LOD is an open-source project and it's maintained by the AKSW group from University of Leipzig. and you can download and deploy the project from our source code. Dynamic-LOD uses Data Driven Documents JS Library to create a cloud-based diagram. A detailed description of this work can be found in this paper.
