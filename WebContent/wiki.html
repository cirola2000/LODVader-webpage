Using our Online demo

We provide a free Online instance of Dynamic-LOD. The API is available at the address http://vmdbpedia.informatik.uni-leipzig.de:9090/dataid/api, and for the diagram visualization, we provide the address http://vmdbpedia.informatik.uni-leipzig.de:9090/dataid/cloud.html?dataset=.

Diagram Overview

Dynamic-LOD provides a diagram which you can add your datasets, and find out whether your dataset contain linkset to other datasets. Linksets are defined as intersections of objects and subjects, which means that links occurs when objects and subjects of distinct datasets are equals. The visualization consists of two parts, which includes a tree of datasets and an area which is updates with a SVG diagram according to the selected datasets of the tree.

The root of the tree contains the top datasets, i.e. datasets which don't contains a parent dataset. The following figure depicts the structure of DBpedia commons datasets:

Notice that the top dataset named "DBpedia 2014 commons dumps dataset" contains several subsets (categories, links, degrees, etc), and each subset contains a dump file (distribution). You can select one or more item to filter and create a customized graph. The following figure depicts the generated diagram for the selected distributions.
That there are three different nodes (bubbles) colors. The grey nodes are the nodes selected from the tree, green nodes the related datasets, and finally the orange nodes are ontologies. The arrows represents the links direction between the nodes. Notice that if your dataset has not been shown in the graph, means the current dataset doesn't have more that 50 links.

API usage

Dynamic-LOD API has three main features, which consists in add new dataset to the diagram, retrieve data about the streaming status, and finally, retrieve RDF and a diagram about related datasets. To add a dataset and retrieve the status, the API returns a JSON Object which contains a "message" field and a "success" filed with a boolean value. The following sections describe thw API usage.

Adding dataset to diagram

Dynamic-LOD parses different description files, such as DataID, VoID and DCAT. Thus, what you need to add your datasets is a link that contains the description file, and Dynamic-LOD will read it and try to fetch distributions dump files. Considering that the application requests header in the format "Accept: application/rdf+xml" it's also possible, for instance, to add your dataset via a CKAN link.

The API parameters used to add a dataset are:

Parameter: Description: Accepts array? Returns:
addDataset link for your dataset description. Might be a list of links. Yes JSON Object with a core message "API initialized" and (later, when the webpage is refreshed) a parser message with the number of distributions parsed.
rdfFormat format of the added links in the addDataset parameter. Formats are: ttl, nt or rdfxml. No

An generic example of a link to add a http://x.y.x/dataset.ttl file in turtle is: vmdbpedia.informatik.uni-leipzig.de:9090/dataid/api?addDataset=http://x.y.x/dataset.ttl?rdfFormat=ttl .

After adding the dataset, you should get a message indicating if the API was successfully initiated. To check details about parsed distributions and datasets, check the next section for the datasetStatus parameter.

Checking your dataset status

The API parameter used to verify the details of the loading/streaming process for a dataset is:

Parameter Description Accepts array? Returns
datasetStatus link for your previouly added dataset. No JSON Object (described in the next table)
A detailed description for the returned JSON Object of the datasetStatus parameter is given by the next table.
Key Description Possible values
downloadURL Identify the distributions which being dexcribed. The distribution URL
success boolean value. Case value is false, an error occured and you can check the "lastErrorMessage" or the "message" value. true or false
lastErrorMessage A friendly message indicating the error case "success" field is false. -
message General message returned from server. -
topDataset Returns the top parent dataset of the described distribution. Top dataset URI
status The current status for the distribution. Described in the next table
Even if you loaded tousands of distributions, the "status" key allows you to check individually the status of each of them. The next table details, in order, each possible "status" value.
Value Description
WAITING_TO_DOWNLOAD Your distribution is in the download queue and was not streamed yet.
DOWNLOADING Your distribution being streamed/downloaded.
CREATING_BLOOM_FILTER Your distribution was successfully streamed and Dynamic-LOD is creating the Bloom filter.
WAITING_TO_CREATE_LINKSETS Shortly, the current distribution will be compared with other to count links between them.
CREATING_LINKSETS Dynamic-LOD is comparing an counting links of the current distribution with distributions previoulsy stored in our server.
DONE The process was complete with no errors.
ERROR Something went wrong and the error is described in the "lastErrorMessage" field.
